# 核心设计模式

> 本文档描述 CodeMind 中的关键设计模式和业务规则。

## 用户角色（三级 RBAC）

| 角色 | 标识 | 权限范围 |
|------|------|---------|
| 超级管理员 | `super_admin` | 全部系统功能 |
| 部门经理 | `dept_manager` | 管理本部门用户和统计 |
| 普通用户 | `user` | 个人 API Key、用量、第三方服务商 |

### 路由与角色对应

- `super_admin`：系统管理、审计日志、服务商模板、平台设置
- `dept_manager`：用户管理（仅本部门）、部门统计
- `user`：个人 API Key、用量查看、第三方服务商绑定

详细权限矩阵参见 [权限控制优化](permission-optimization.md)。

## API Key 格式

- **格式**：`cm-{32-char-hex}`
- **安全**：仅在创建时展示一次完整密钥，数据库存储 SHA-256 哈希值
- **前缀**：存储 `cm-` 前缀的前 12 个字符用于界面展示标识

## 速率限制优先级

用户级限额 > 部门级限额 > 全局限额

当用户同时受多个层级限制时，取最严格的限制值。

## 软删除

所有数据模型使用 `deleted_at` 时间戳实现 GORM 软删除模式。删除操作不会物理移除数据，而是标记删除时间。

## 登录锁定

失败的登录尝试触发指数退避锁定：
- 首次失败：5 分钟
- 持续失败：时间指数增长
- 最大锁定时间：24 小时
